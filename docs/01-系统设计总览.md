## 一、项目概述

### 1.1 项目背景

这是一个面向教育培训机构的学习管理系统（LMS），支持：
- 多角色用户（学区管理员、系统管理员、教师、学生、家长、招生合作伙伴）
- 课程创建与管理
- 作业与评估系统
- 学习进度跟踪
- 标准对齐（州/省教育标准）

### 1.2 分期规划

| 阶段 | 目标 | 核心功能 | 预计时间 |
|------|------|----------|----------|
| **一期** | 老师可用的MVP | 权限管理、课程管理、作业评估、基础报表 | 6-8周 |
| **二期** | 可招生、可运营 | 测验考试、评分标准、运营报表 | 4-6周 |
| **三期** | 体验提升 | 移动端优化、第三方集成 | 4-6周 |
| **待评估** | AI增强 | AI内容生成、视频反馈 | 视评估而定 |

## 四、核心业务流程

### 4.1 用户认证流程

```
用户 → 输入邮箱/密码
    → 后端验证
    → 生成JWT Token
    → 返回Token + 用户信息
    → 前端存储Token
    → 后续请求携带Token
    → 后端验证Token
    → 返回数据
```

### 4.2 课程创建流程（教师）

```
教师 → 选择课程模板（可选）
    → 填写课程基本信息
    → 使用拖拽编辑器组织课程结构
    → 添加内容模块（视频、文档、测验等）
    → 关联教育标准（可选）
    → 设置发布规则（立即/定时/条件）
    → 保存草稿/发布课程
```

### 4.3 作业提交流程（学生）

```
学生 → 查看作业列表
    → 选择作业
    → 查看作业详情和要求
    → 完成作业（在线编辑/文件上传）
    → 提交作业
    → 系统记录提交时间
    → 通知教师
    → 教师评分/反馈
    → 通知学生
```

### 4.4 权限检查流程

```
请求 → 携带JWT Token
    → AuthGuard验证Token
    → 解析用户信息
    → RolesGuard检查角色
    → PermissionsGuard检查权限
    → 允许/拒绝访问
```

## 五、安全设计

### 5.1 认证与授权

- **JWT Token认证**：无状态、可扩展
- **RBAC权限模型**：角色-权限-资源
- **细粒度权限控制**：功能级、课程级
- **密码安全**：bcrypt加密、密码强度要求

### 5.2 数据安全

- **HTTPS加密传输**
- **敏感数据加密存储**
- **SQL注入防护**：Prisma参数化查询
- **XSS防护**：输入验证、输出转义
- **CSRF防护**：Token验证

### 5.3 API安全

- **Rate Limiting**：防止暴力攻击
- **API Key管理**：第三方集成
- **日志审计**：关键操作记录

## 六、性能优化

### 6.1 前端优化

- **代码分割**：React.lazy + Suspense
- **资源懒加载**：图片、视频懒加载
- **缓存策略**：React Query缓存
- **CDN加速**：CloudFront

### 6.2 后端优化

- **数据库索引**：关键字段索引
- **查询优化**：避免N+1查询
- **Redis缓存**：热点数据缓存
- **分页查询**：大数据集分页
- **连接池**：数据库连接池

### 6.3 文件优化

- **图片压缩**：上传时自动压缩
- **视频转码**：H.264编码
- **断点续传**：大文件上传
- **CDN分发**：CloudFront
